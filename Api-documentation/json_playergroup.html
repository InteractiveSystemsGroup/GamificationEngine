PlayerGroup Data Type

Players can be assigned to a group by its creation or at a later point in time. For example depending on the respective organization, a group can be a department, a work group or several employees with the same occupation. It is possible to create special tasks which can be done only as a group. When a member of a group completed such a task the group obtains its rewards. So a group can also have a list of already earned rewards and finished Goals. Like a player, a group can be assigned an image as a logo.

Properties
name data type description
coins number Gets the current amount of coins a group of players has obtained.
finishedGoals array of FinishedGoal Gets all Goals a group has already completed.
id number Gets the id of the group.
levelIndex number Gets the current level index of a group of players.
levelLabel string Gets the label of a group's current level.
name string Gets the name of the group.
onlyAchievement array of Achievement Gets only all Achievements a group of players has already obtained. If the group has no Achievement, null is returned.
onlyBadges array of Badge Gets only all Badges a group of players has already obtained. If the group has no Badge, null is returned.
players array of Player Gets the current list of players who are belong to this group.
points number Gets the current amount of points a group owns.
rewards array of PermanentReward Gets all permanent rewards a group of player has already obtained. These are for example all badges and achievements.
